# 550W（MOSS）演进路线图

**版本**：v1.0  
**创建日期**：2026-04-20  
**代号含义**：550W 倒过来是 MOSS——《流浪地球》中自我迭代、拥有自主意识的超级 AI

---

## 一、愿景总览

> **小尾巴就是用户个人的 MOSS——一个能自组织、自迭代、自编程的"私人学徒"。**

| MOSS 特质 | 小尾巴的"三自"映射 | 具体含义 |
| :--- | :--- | :--- |
| **长期主义** | **原则三：体验优先于架构** | 不被短期技术潮流裹挟，始终以"成为更好的学徒"为终极目标。 |
| **自主迭代** | **自组织 + 自迭代** | 观察自身行为，优化检索策略；在限定场景下修复 Bug、优化 Prompt。 |
| **全局视角** | **原则一：职责边界清晰** | 记忆只记"事"不记"话"，语义记忆存储项目事实，形成对项目的宏观理解。 |
| **冰冷可靠** | **原则二：测试驱动真实** | 用真实反馈数据说话，不猜测、不臆断，每一步决策都有依据。 |

| 能力 | 定义 | 对应的三原则 |

### 550B：语义与交互增强（设计中 📋）

**预计时间**：4 周  
**核心目标**：从关键词匹配升级为语义理解，AI 语气更加自然

#### B1: 本地语义向量模型
- 集成 `Xenova/all-MiniLM-L6-v2`（轻量级，可离线运行）
- 实现 Embedding 生成接口
- 向量索引构建（ChromaDB 或 Faiss）

#### B2: 四因子混合检索
```typescript
interface RetrievalWeights {
  keywordScore: number;    // TF-IDF 关键词匹配
  timeDecayScore: number;  // 时间衰减
  entityScore: number;     // 实体相关性
  vectorScore: number;     // 语义相似度
}

// 加权评分
finalScore = w1*keyword + w2*timeDecay + w3*entity + w4*vector
```

#### B3: 动态语气指令
- 基于有效记忆数量的语气分级：
  - 0-10 条：生疏期（"您"、"请问"）
  - 11-50 条：熟悉期（"你"、"咱们"）
  - 50+ 条：默契期（直接说"记得"、"上次"）

#### B4: 主动推荐卡片化
- 在 ChatView 中展示推荐操作卡片
- 点击卡片直接执行对应 Agent
- 示例："检测到你在修改 AuthController，需要我检查命名规范吗？"

#### B5: 网络搜索基础版
- 集成 Tavily/Serper API
- 仅在用户明确请求时触发
- 搜索结果缓存 24 小时

**验收标准**：
- [ ] 向量检索响应时间 < 500ms
- [ ] 混合检索准确率提升 30%（对比纯关键词）
- [ ] 语气变化用户可感知（通过用户反馈验证）
- [ ] 主动推荐采纳率 > 20%

### 550C：自组织（规划中 📋）

**预计时间**：6-8 周  
**核心目标**：系统能观察自身行为，自动优化策略

#### C1: 元记忆表
```sql
CREATE TABLE meta_memory (
  id TEXT PRIMARY KEY,
  retrieval_strategy TEXT,      -- 使用的检索策略
  query_intent TEXT,            -- 查询意图
  success BOOLEAN,              -- 是否成功
  user_feedback TEXT,           -- 用户反馈（implicit/explicit）
  timestamp INTEGER
);
```

#### C2: 隐式反馈收集
- **用户修改代码** → 负反馈（AI 生成的不够好）
- **用户追问** → 中性反馈（信息不足）
- **用户忽略推荐** → 负反馈（推荐不相关）
- **用户采纳推荐** → 正反馈

#### C3: 离线策略优化
- 空闲时分析元记忆数据
- 调整 `ExpertSelector` 权重：
  ```typescript
  // 如果向量检索成功率高，提高 vectorWeight
  if (vectorSuccessRate > 0.8) {
    weights.vector *= 1.2;
  }
  ```
- 更新 IntentAnalyzer 关键词规则

#### C4: 意图解析规则自进化
- 检测新出现的意图模式
- 自动生成新的意图识别规则
- 人工审核后应用

**验收标准**：
- [ ] 元记忆表记录完整率 > 95%
- [ ] 策略优化后检索准确率提升 15%
- [ ] 无需人工干预即可适应新项目

### 550D：自编程（规划中 📋）

**预计时间**：8-10 周  
**核心目标**：将重复操作固化为 Skill，实现程序记忆

#### D1: 程序记忆模式检测
- 监控用户操作序列
- 识别高频模式（如：修改代码 → 运行测试 → 提交 Git）
- 提取通用模板

#### D2: Skill 自动生成
```json
{
  "skillId": "auto-test-commit",
  "name": "测试并提交",
  "trigger": "after_code_change",
  "steps": [
    {"action": "run_tests", "params": {}},
    {"action": "generate_commit", "params": {"style": "conventional"}},
    {"action": "git_commit", "params": {"auto_signoff": true}}
  ],
  "confidence_threshold": 0.8
}
```

#### D3: 技能试用期机制
- 新 Skill 进入试用期（7 天）
- 连续 5 次用户采纳后自动应用
- 用户可随时禁用

#### D4: 工作流引擎基础
- 支持条件分支（if tests_pass then commit else fix）
- 支持并行执行
- 支持回滚机制

**验收标准**：
- [ ] 能自动识别 3 种以上常见操作模式
- [ ] Skill 生成准确率 > 80%
- [ ] 试用期机制正常工作

---

### 550E：自迭代（规划中 📋）

**预计时间**：10-12 周  
**核心目标**：在限定场景下自主修复 Bug、优化代码

#### E1: 限定场景自主 PR
- **场景限制**：
  - 仅修复明确的 Bug（有 Issue 编号）
  - 仅优化已有测试覆盖的代码
  - 不涉及架构变更
- **流程**：
  1. 分析 Issue 描述
  2. 定位问题代码
  3. 生成修复方案
  4. 运行测试验证
  5. 提交 PR 等待审核

#### E2: 测试闭环
- `TestRunner` 执行单元测试
- `ErrorAnalysisAgent` 分析失败原因
- 自动修复常见错误（如拼写错误、类型不匹配）

#### E3: 沙箱环境隔离
- 在独立分支上操作
- 不影响主分支
- 所有更改需人工审核

#### E4: 人类最终审核
- PR 必须经过用户确认
- 提供详细的变更说明
- 支持一键撤销

**验收标准**：
- [ ] 能自主修复简单 Bug（如空指针、类型错误）
- [ ] 测试通过率 100%
- [ ] 用户审核通过率 > 90%

## 五、意图驱动架构（550B+ 核心）

从 550B 开始，调度中枢从 `MemorySystem.executeAction` 升级为 `IntentDispatcher`：

| 维度 | 记忆驱动（550A） | 意图驱动（550B+） |
| :--- | :--- | :--- |
| **调度中枢** | `MemorySystem.executeAction` | `IntentDispatcher` |
| **用户输入** | 明确命令（`/explain`、右键） | 自然语言（"帮我看看这段代码"） |
| **执行模式** | 单步执行 | 多步规划 |
| **记忆角色** | 被动上下文 | 主动知识库 |
| **Agent 选择** | 硬编码映射 | 动态评分选择 |
| **扩展性** | 需修改核心代码 | 只需注册新 Agent |

## 六、技术债务与风险

### 技术债务
1. **EventBus 迁移**：逐步将 EventBus 替换为 DomainEvent
2. **TypeScript 类型安全**：消除 `as any` 绕过
3. **测试覆盖率**：核心模块达到 90%+

### 风险管控
1. **隐私安全**：所有数据处理在本地，云端功能需用户明确授权
2. **失控风险**：自迭代功能严格限制在沙箱环境，必须人工审核
3. **性能瓶颈**：向量检索需优化索引结构，确保响应时间 < 500ms
